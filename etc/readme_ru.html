Импорт данных с формате csv

Для импорта и обновления данных вам необходимо создать два файла: файл данных в формате csv и mapping-файл. Например, для импорта пользователей создадим файл данных в формате csv csvuser.csv содержащий данные подобно этим:

Login;Name;Status;Email
peter;Peter Johnson;Developer;developer@localhost
john;John Peretson;Manager;manager@localhost
simba;Simba Toowe;Tester;tester@localhost
и mapping-файл csvuser.properties подобно следующему:
PRSTATUS=Status
LOGIN=Login
USER_NAME=Name
EMAIL=Email
Где, имена с левой стороны - это именя полей TrackStudio, а имена в правой стороны это поля имен содержащиеся в файле данных csv.

Примечание 1: Имена полей в mapping-файле чувствительны к регистру; имена должны точно соответствовать полям TrackStudio и распологатся на одной строке одно поле-имя.
Примечание 2: Имена полей в файле данных могут содежрать поля которых нет в mapping-файле. Используется для импорта дополнительных полей.
Примечание 3: Файл данных должен иметь расширением .csv, а mapping-файл должен иметь расширение .properties. Хотя файлы с другими расширениями могут быть так же использованы и выбраны.
Примечание 4: При выборе имен полей для импорта пользователей вы можете использовать произвольные названия. Они могут содержать пробелы, но их необходимо конвертировать в соответствии с mapping-файлом.
Примечание 5: В том случае если имена полей или данные содежат пробелы их необходимо заменить обратным слешом. пример: Section\Membership\(ACD\ Type)=Section\ Membership
Примечание 6: Если вы хотите узнать какие поля необходимо использовать для вашего mapping-file, Вы можете экпортировать соответствующие поля из TrackStudio и посмотреть первую колонку в файле отчета типа CSV.

Другие названия полей TrackStudio распологаются в файле demo/cvsdefault.properties
