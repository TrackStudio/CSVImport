Импорт данных с формате csv

Для импорта и обновления данных вам необходимо создать два файла: файл данных в формате csv и mapping-файл. Например, для импорта пользователей создадим файл в формате csv csvuser.csv содержащий данные:

Login;Name;Status;Email
peter;Peter Johnson;Developer;developer@localhost
john;John Peretson;Manager;manager@localhost
simba;Simba Toowe;Tester;tester@localhost
и mapping-файл csvuser.properties:
PRSTATUS=Status
LOGIN=Login
USER_NAME=Name
EMAIL=Email
ключи с левой стороны - это имена полей TrackStudio, а значения в правой стороне - это заголовки полей, содержащиеся в файле данных csv.

Примечание 1: Названия полей в mapping-файле чувствительны к регистру; они должны точно соответствовать полям TrackStudio.
Примечание 2: Названия полей в файле данных могут содержать поля, которых нет в mapping-файле. В таком случае они импортируются как дополнительные поля TrackStudio.
Примечание 3: Файл данных должен иметь расширение .csv, а mapping-файл - .properties.
Примечание 4: В случае, если названия полей в мэппинге (ключи или значения) содежат пробелы, перед пробелом следует поставить обратный слеш. пример: Section\Membership\(ACD\ Type)=Section\ Membership
Примечание 5: Если вы хотите узнать, какие поля необходимо использовать для вашего мэппинг-файла, вы можете построить отчет типа "Список" по любым задач в TrackStudio в формате CSV. Названия стандартных полей TrackStudio располагаются в файле demo/cvsdefault.properties
